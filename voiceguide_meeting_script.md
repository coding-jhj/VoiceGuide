## 1. 프로젝트 한 줄 소개

저희 프로젝트 보이스가이드는,
**시각장애인 또는 저시력 사용자가 음성 명령과 카메라를 통해 주변 장애물, 찾는 물건, 손에 든 물건을 확인할 수 있도록 돕는 TTS-STT 기반 주변 인지 보조 서비스**입니다.

목표는 **10일 안에 작동하는 데모를 완성**하는 것이며, 이후 6월 1일 국민행복 서비스 발굴 공모전 제출까지 연계할 계획입니다.

---

## 2. 핵심 기능 — 딱 3가지만 합니다

데모에서 보여줄 기능은 딱 3가지입니다.

| 기능 | 음성 입력 | 시스템 출력 |
|---|---|---|
| 장애물 안내 | "앞에 뭐 있어?" | "정면 가까이에 의자가 있습니다." |
| 물건 찾기 | "컵 찾아줘" | "컵은 왼쪽 앞에 있습니다." |
| 물건 확인 | "이거 뭐야?" | "카메라 중앙의 물건은 텀블러입니다." |

욕심을 줄이고 이 3가지를 완벽하게 작동시키는 것에 집중합니다.

---

## 3. 기술 스택

다음은 사용하는 기술 스택입니다.

| 역할 | 기술 | 선택 이유 |
|---|---|---|
| 객체 탐지 | YOLO11n | 추가 학습 없이 80종 인식, 설치 한 줄 |
| 방향 판단 | bounding box 좌표 계산 | 라이브러리 불필요 |
| 거리 판단 | bounding box 면적 비율 | Depth 모델 없이 대체 |
| 의도 파악 | 키워드 매칭 if/elif | LLM 없이도 빠르고 안정적 |
| TTS | gTTS + pygame | 한국어 발음 자연스러움 |
| STT | SpeechRecognition | 한국어 지원, 불안정 시 버튼으로 대체 |
| UI | Gradio | Python만으로 웹 화면 자동 생성 |

특히 **모델 학습이나 LLM은 사용하지 않습니다.** YOLO는 사전학습 모델을 그대로 사용하고, 의도 파악은 키워드 매칭으로 처리합니다. 발표장 인터넷 상황에 상관없이 안정적으로 작동하도록 하기 위해서입니다.

## 4. 역할 분담

다음은 6명의 역할 분담입니다.

| 역할 | 담당 업무 |
|---|---|
| **구현 A** | YOLO11n 설치 + 객체 탐지 + 방향/거리 판단 |
| **구현 B** | TTS + STT + 의도 파악 (키워드 매칭) |
| **구현 C** | 시나리오 3개 통합 + Gradio UI |
| **기획/자료 (2명)** | 문제 정의, 자료조사, PPT 제작, 발표 |
| **테스트/데모** | 테스트 시나리오 설계, 결과 기록, 데모 영상 촬영 |

구현팀은 파일 단위로 역할을 나눴습니다. `detect.py`, `voice.py`, `app.py` 각자 담당 파일만 작업하고, 구현 C가 최종 통합합니다.

## 5. 온라인 팀플 진행 방식

온라인으로 진행하다 보니 환경 차이와 통합 타이밍이 리스크입니다. 이렇게 대응할 계획입니다.

**첫째, 환경을 완전히 통일합니다.**
Python 3.10으로 버전을 맞추고, `requirements.txt`에 패키지 버전을 전부 고정합니다.
팀원 전원이 clone 후 `pip install -r requirements.txt` 한 줄로 동일 환경을 세팅합니다.

**둘째, Git 브랜치로 충돌을 방지합니다.**
`main` 브랜치는 항상 돌아가는 버전으로 유지하고, 각 구현 담당자는 `feature/detect`, `feature/voice`, `feature/app` 브랜치에서 작업합니다. VS Code에서 브랜치 전환 및 병합을 관리합니다.

**셋째, 웹캠 대신 이미지 파일로 테스트합니다.**
`test_images/` 폴더에 테스트용 이미지를 GitHub에 올려두고, 팀원 전체가 동일한 이미지로 테스트합니다. 실시간 카메라 없이도 전체 파이프라인 테스트가 가능합니다.

**넷째, 발표는 데모 영상으로 대체합니다.**
실시간 데모 대신 시나리오 3개를 사전 녹화하여 제출합니다. 발표 당일 네트워크 상황에 상관없이 안정적으로 시연할 수 있습니다.

## 6. 10일 일정

| 일차 | 구현팀 | 기획/자료팀 | 테스트/데모팀 |
|---|---|---|---|
| 1일 | YOLO11n 설치 + 이미지 파일로 테스트 | 문제 정의 확정, 자료조사 시작 | 테스트 시나리오 설계, 공용 이미지 수집 |
| 2일 | 물체 인식 확인, test_images 업로드 | WHO/보건복지부 자료 정리 | 테스트 물체 선정, 성공 기준 작성 |
| 3일 | 방향/거리 판단 로직 완성 | 기존 서비스 비교표 작성 | 테스트 결과 기록표 초안 |
| 4일 | gTTS 연결, 음성 출력 확인 | 차별점 3개 정리, PPT 구조 확정 | 구현팀 보조 |
| 5일 | STT 연결 or 버튼 대체 결정 | PPT 본문 작성 시작 | 구현팀 보조 |
| 6일 | 시나리오 3개 통합 시작 | PPT 본문 작성 | Gradio 링크로 1차 테스트 |
| 7일 | Gradio UI 완성, 팀 전체 테스트 | PPT 완성 | 2차 테스트, 실패 사례 정리 |
| 8일 | 오류 수정, 잘 되는 물체만 데모 사용 | 발표 대본 작성 | 테스트 결과표 최종 완성 |
| 9일 | 최종 오류 수정 | 리허설 준비 | 데모 영상 녹화 |
| 10일 | 리허설 참여 | 발표 리허설 2회 이상 | 영상 확인 + 제출 파일 정리 |

**7일차 체크포인트:** 시나리오 3개가 작동하지 않으면 STT 제거 → 버튼 대체, 인식 물체 종류 축소, 거리 안내 제거 순으로 범위를 줄입니다.
