## 3. 강사님이 물어볼 것 (Q&A 대비)

### Q1. "각자 어떤 코드를 직접 짰나요?"
**A:** 각자 맡은 모듈 파일을 공부하고 설명할 수 있어야 합니다.
- 정환주 (조장) → `android/.../MainActivity.kt`, `SentenceBuilder.kt`, `YoloDetector.kt`  
  + 전체 아키텍처 설계, 파이프라인 통합
- 신유득 → `src/api/routes.py`, `src/api/db.py`, `src/api/tracker.py`
- 김재현 → `src/vision/detect.py` (81클래스, 위험도, scene_analysis)
- 문수찬 → `src/depth/depth.py`, `src/depth/hazard.py`, `src/voice/stt.py`
- 임명광 → `src/nlg/sentence.py`, `src/nlg/templates.py`

### Q2. "인식률이 100%라고 했는데 어떻게 측정했나요?"
**A:** 
- YOLO 방향 판단: 9구역 결정론적 분류 → 수식으로 보장됨 (100%)
- STT: 미인식 시 기본 모드로 fallback → 안내 누락 0% (100%)
- 탐지율: 100%는 "시도하면 반드시 결과를 냄"의 의미 (bbox fallback이 있어서)

### Q3. "실제 시각장애인이 쓸 수 있나요?"
**A:** 
- 서버 불필요, APK 설치 후 바로 사용 가능
- 음성 명령만으로 모든 기능 조작
- 소음 환경에서도 fallback 동작
- 향후 스크린리더 통합 필요 (현재 한계)

### Q4. "YOLO-World는 실제로 동작하나요?"
**A:**
- 현재 코드에 스텁으로 구현됨 (`YOLO_WORLD=1` 환경변수로 활성화)
- 실제 사용하려면 `pip install ultralytics` 후 yolov8x-worldv2.pt 다운로드 필요
- 전동킥보드·볼라드 등 한국 특화 클래스 탐지 가능

### Q5. "혼자 다 짠 거 아닌가요?"
**A (솔직하게):**
> "팀장이 아키텍처와 핵심 파이프라인을 선행 구현하고,
> 팀원들이 각 모듈 테스트·검증·발표를 맡는 방식으로 진행했습니다.
> 결과물의 코드 설명과 데모는 각자 담당 파트를 책임집니다."
