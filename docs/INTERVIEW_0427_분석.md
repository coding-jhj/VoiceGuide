## Q&A 전문 + 개발팀 해석

### 1부. 일상 이동 경험

**Q1. 주로 사용하는 이동 수단**
- 답: 택시
- 해석: 대중교통 접근성 어려움 → 버스 번호 감지(Q8), 하차 알림 기능이 실제 필요

**Q2. 가장 불안한 순간**
- 답: 처음 가는 곳
- 해석: 개인 네비게이팅(장소 저장/찾기)이 핵심 기능임을 확인 ✅

**Q3. 점자 블록 경험**
- 답: 도움되나 킥보드·자전거 등 장애물이 많아 중단됨
- 해석: 점자 블록 위 장애물 감지 기능(이미 구현 ✅) 실사용 가치 확인

**Q4. 버스·지하철 어려운 점**
- 답: 사람이 많고 안내판 부족
- 해석: 군중 경고(3명+, ✅ 구현)와 안내판 OCR 읽기가 실용적

**Q5. 키오스크**
- 답: 타인 도움 없이 절대 사용 불가
- 해석: 텍스트 읽기 기능이 키오스크 화면에도 적용되면 가치 있음

### 3부. 앱 기능 의견

**Q10. 방향+행동 안내 유용성**
- 답: "자주 말해주면 오히려 불편. 약 5m 앞에 장애물이 있음 같은 대략적 설명 필요"
- 해석: 
  - ✅ VotingBuffer(10프레임 다수결)로 과도한 경고 이미 억제
  - ✅ `_format_dist` 상대 표현("가까이", "멀리") 사용 중
  - 📌 거리 표현 개선 검토: "5미터 앞" vs "조금 멀리" — Depth V2 있으면 수치도 가능
  - 서버 `get_alert_mode` critical 범위를 2.5m로 확장 ✅

**Q11. 비프음 vs 음성 설명**
- 답: "비프음보다는 말로 설명하는 것이 편함"
- 해석:
  - Android: 비프음(TONE_PROP_BEEP2) → 음성 안내로 변경 ✅
  - 서버: `get_alert_mode` "beep" 단계 제거, critical 범위 확장 ✅

**Q12. 개인 네비게이팅**
- 답: "몹시 도움이 될 것 같음"
- 해석: ✅ 이미 구현 완료. 강력한 차별 기능 확인

**Q13. 신호등 감지**
- 답: "카메라로 신호등을 찾기가 어려울 것 같음"
- 해석:
  - 현재: 사용자가 "신호등" STT 명령 → 카메라 직접 겨냥 필요
  - 개선 방향: 교차로 근처 자동 감지 (GPS + 신호등 자동 탐지)
  - 📌 Q17 원칙("카메라 정밀 조작 금지")과 연결 — 자동 감지로 해결해야 함
  - 단기: 현재 구현 유지, 발표 시 한계 솔직하게 인정

**Q14. 글자 읽기(OCR)**
- 답: "절실함"
- 해석:
  - ✅ "글자 읽어줘" STT 모드 구현 완료
  - 📌 정확도 측정 필요: 간판, 메뉴판, 처방전 등 실환경 테스트

**Q15. 가장 중요한 것**
- 답: "정확도"
- 해석: 오탐/미탐 모두 신뢰 손상. VotingBuffer + IoU dedup + Depth V2가 핵심
