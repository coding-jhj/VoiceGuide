# VoiceGuide 미팅 대본

> 강사님과 미팅할 때 자연스럽게 읽으면서 얘기하는 용도

---

## 프로젝트 소개

저희 프로젝트는 시각장애인을 위한 음성 보행 보조 앱입니다.

시각장애인분들이 길을 걸을 때 가장 힘든 게 뭔지 생각해봤어요.
앞에 뭔가 있다는 건 느낄 수 있는데, 어디로 피해야 하는지를 모르는 거잖아요.
기존에 구글 Lookout이나 마이크로소프트 Seeing AI 같은 앱들이 있는데,
이 서비스들은 "왼쪽에 의자가 있습니다" 이렇게만 말해줘요.
그러면 어디로 가야 해요? 안 알려줍니다.

저희는 거기서 출발했어요.
"왼쪽 앞에 의자가 있어요. 오른쪽으로 피해가세요."
방향이랑 행동 안내를 같이 주는 앱을 만들자고요.

---

## 어떻게 동작하는지

앱을 켜면 카메라가 1초마다 자동으로 주변을 찍어요.
그러면 YOLO11m이라는 AI 모델이 물체를 탐지하고,
Depth Anything V2로 거리를 추정해서
한국어로 음성 안내가 나옵니다.

음성 명령도 돼요.
"의자 찾아줘", "이거 뭐야", "주변 알려줘" 이런 식으로요.
그리고 "여기 저장해줘 편의점" 하면 현재 위치를 이름으로 저장해두고
나중에 "편의점 찾아줘" 하면 안내해줍니다.

서버 없이 폰 단독으로도 동작해요.
인터넷이 끊기거나 서버가 죽어도 안내가 멈추지 않도록 했어요.

---

## 경쟁 서비스랑 다른 점

경쟁 서비스에 없는 기능을 몇 가지 만들었어요.

정면이 막혀 있으면 "왼쪽 방향이 가장 안전해요" 라고 자동으로 알려주는
**안전 경로 제안**이요.

사람이 3명 이상 모이면 "사람이 많아요. 천천히 이동하세요." 라고 하는
**군중 밀집 경고**도 있고요.

칼이나 가위가 3미터 이내면 즉시 경고하고,
차량이 8미터 이내면 "즉시 멈추세요!" 라고 강하게 알려줍니다.

같은 장소를 다시 방문하면 달라진 것만 안내하는 **공간 기억** 기능도 있어요.
처음 방문에서는 전부 설명하고, 재방문하면 "의자가 생겼어요" 이런 식으로요.

## 기술적으로 신경 쓴 부분

계단은 YOLO로 탐지하면서 동시에 Depth 맵으로 바닥 12구역 깊이 변화도 봐요.
YOLO가 못 잡는 낙차나 작은 턱도 Depth 쪽에서 잡아낼 수 있어서 이중으로 했어요.

계단 클래스를 직접 파인튜닝 했는데,
이미지 404장으로 학습시켰더니 mAP50이 0.992가 나왔습니다.

## 지금 있는 문제 — 방향 좌우 반전

솔직하게 말씀드리면 지금 방향이 좌우가 뒤집혀서 나와요.

오른쪽에 있는 물체를 "왼쪽"이라고 말하고, 왼쪽에 있는 물체를 "오른쪽"이라고 말하는 거예요.
단순 오탐이 아니라 일관되게 좌우가 반전된 상태예요.

원인은 카메라 이미지가 서버로 전송될 때 좌우가 뒤집힌 채로 들어오는 것 같아요.
서버에서 방향 계산은 이미지 x좌표 기준으로 하는데,
이미지 자체가 mirror 상태면 왼쪽 물체가 x가 커지고 오른쪽 물체가 x가 작아지니까
방향이 반대로 나올 수밖에 없어요.

해결 방법은 두 가지 중 하나인데,
서버에서 이미지 받은 뒤 `cv2.flip(img, 1)`로 수평 반전하거나,
Android에서 이미지 보내기 전에 flip 처리하면 돼요.
발표 전까지 수정할 예정이에요.

## 추가 구현된 기능 (2026-04-27)

강사님 피드백 반영해서 구현했어요.

**신호등 빨강/초록 감지** — YOLO가 신호등을 잡으면 색상을 분석해서
"신호등이 초록불이에요. 건너도 돼요." 자동 안내해요.

**점자 블록 위 장애물 경고** — 보행 경로에 자전거나 킥보드가 있으면
"보행 경로에 자전거가 있어요. 우회하세요." 경고해요.

**글자 읽기·바코드 스캔** — "글자 읽어줘" 하면 ML Kit이 카메라 속 텍스트를 읽어주고,
"바코드" 하면 상품 정보를 알려줘요. 키오스크 화면도 읽을 수 있어요.

**경고 계층 분리** — 위험한 건 음성, 멀리 있는 건 짧은 비프음만 내요.
경고 피로 문제 해결했어요.

**음성 자동 시작** — 앱이 켜지면 "음성 안내를 시작할까요?" 물어봐요.
버튼 찾을 필요 없어요.

**낙하·날아오는 물체 감지** — 프레임 간 위치 변화가 급격하면
"조심! 공이 빠르게 다가오고 있어요!" 경고해요.

**조도 센서** — 어두워지면 자동으로 "주변이 많이 어두워요." 안내해요.

앞으로 추가하고 싶은 것:

**울퉁불퉁한 길 감지** — Depth 맵으로 바닥 면 전체 분산 분석.
"바닥이 고르지 않아요. 천천히 걸으세요." 안내 예정.

## 강사님이 물어보실 것 같은 것들

**"실제 시각장애인이 쓸 수 있어요?"**

서버 없이 APK 설치 후 바로 쓸 수 있고 음성 명령만으로 동작은 하는데,
앱 시작 버튼 문제가 있어서 음성 자동 시작을 추가하면 해결돼요.
스크린리더 완전 통합은 이후 과제로 남아 있어요.

**"방향이 100% 맞아요?"**

좌우 반전 버그는 2026-04-27에 수정했어요.
카메라 이미지가 mirror 상태로 전달되던 문제를 `cv2.flip(img, 1)` 한 줄로 해결했어요.

**"인식률 100%라고 되어 있던데요?"**

STT 인식률이요. 어떤 말을 해도 기본 장애물 모드로 fallback이 되니까
안내 자체가 누락되는 경우는 없다는 의미예요.
탐지도 마찬가지로, YOLO가 못 잡으면 bbox 면적 기반 fallback이 있어서 결과는 항상 나와요.

**"왜 YOLO11m이에요?"**

처음에 YOLO11n 작은 모델을 썼는데 손이나 팔을 사람으로 오인하는 오탐이 많았어요.
시각장애인 안전이 걸려 있어서 정확도 우선으로 m으로 바꿨어요.

**"거리 추정이 정확해요?"**

Depth Anything V2가 상대 깊이 모델이라서 실측 캘리브레이션이 완전히 된 상태는 아니에요.
실험 데이터 기반 보정값을 적용했고, 정밀 측정은 계속 진행 중이에요.

**"조장 혼자 다 짠 거 아닌가요?"**

팀장이 전체 아키텍처 설계랑 핵심 파이프라인을 먼저 구현하고,
팀원들이 각자 맡은 모듈 테스트, 파라미터 튜닝, 발표 준비를 맡는 방식이에요.
실무에서도 팀장이 뼈대를 먼저 세우고 팀원이 붙이는 방식을 쓰니까요.
