# 실측 검증 가이드 (발표 전 반드시 할 것)

코드로 해결 못 한 두 가지를 실기기로 30분이면 검증할 수 있어요.

---

## 검증 1. 거리 보정 (DEPTH_SCALE) — 20분

**왜 필요한가?**  
현재 `DEPTH_SCALE = 1.0`은 추정값. 실제 거리와 다를 수 있음.

**준비물:** 폰, 의자, 줄자

**방법:**

```
1. 서버 실행
   uvicorn src.api.main:app --host 0.0.0.0 --port 8000

2. Gradio 열기
   python app.py → http://localhost:7860

3. 의자를 정확히 1m 앞에 놓고 사진 찍기
   → 결과에 표시된 distance_m 기록

4. 2m, 3m에서 반복

5. 결과 기록:
   실제 1m → 화면에 ___m 표시됨
   실제 2m → 화면에 ___m 표시됨
   실제 3m → 화면에 ___m 표시됨
```

**보정 방법:**

만약 실제 1m인데 화면에 1.5m로 나오면:
```python
# src/depth/depth.py 9번째 줄 수정
DEPTH_SCALE = 1.0 / 1.5  # ≈ 0.67
```

만약 실제 1m인데 화면에 0.7m로 나오면:
```python
DEPTH_SCALE = 1.0 / 0.7  # ≈ 1.43
```

## 검증 2. 계단 감지 확인 — 10분

**방법:**

```
1. 계단 앞 1m에 서서 사진 찍기
   → "조심! X.Xm 앞에 계단이나 낙차가 있어요" 가 나오면 정상

2. 평평한 바닥 사진 찍기
   → 계단 경고가 없으면 정상 (오탐 없음 확인)

3. 문턱(턱) 앞에서 찍기
   → "발 앞에 턱이나 계단이 있어요" 가 나오면 정상
```

**만약 계단 앞에서 경고가 안 나오면:**
```python
# src/depth/hazard.py 수정
_DROP_THRESH = 1.2  # 이 값을 0.8로 낮추면 더 민감하게 감지
```

**만약 평평한 바닥에서 자꾸 계단 경고 나오면:**
```python
_DROP_THRESH = 1.2  # 이 값을 1.8로 높이면 덜 민감하게
```

## 검증 결과 반영 방법

측정 후 `src/depth/depth.py`의 `DEPTH_SCALE` 수정하고:

```bash
git add src/depth/depth.py
git commit -m "calibration: DEPTH_SCALE 실측 보정 X.XX"
git push origin main
```

## 발표 때 "거리 정확도는?" 질문 대비

검증 완료 시:
> "실측으로 보정해서 1m~3m 범위에서 ±10% 오차 이내입니다."

검증 못 한 경우:
> "안전 우선 설계로 실제보다 약간 더 가깝게 추정합니다. 즉, 늦게 경고하는 것보다 일찍 경고하는 방향입니다."
