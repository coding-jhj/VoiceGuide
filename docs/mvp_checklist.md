# VoiceGuide MVP 정의서 및 체크리스트

## MVP 목표

시각장애인이 스마트폰 카메라로 주변을 찍으면, AI가 장애물의 **시계 방향·거리·위험도**를 음성으로 안내한다.

## MVP 완료 기준

> 의자 1개가 있는 실내 사진 업로드 → "멈추세요! 의자가 12시 방향 약 0.6m에 있어요." 음성 출력

---

## 기능 체크리스트

### 핵심 파이프라인
- [x] YOLO11n 장애물 탐지 (6종: 사람·의자·테이블·가방·가방(캐리어)·휴대폰)
- [x] 방향 판단 — 이미지 9구역, 시계 방향(8시~4시) 출력
- [x] 거리 추정 — bbox 면적 비율 기반, 5단계(매우 가까이~매우 멀리)
- [x] 거리 수치 출력 — 약 X.Xm (보정 상수 CALIB_RATIO/CALIB_DIST_M)
- [x] 위험도 계산 — 방향·거리 가중치 곱, 내림차순 정렬
- [x] 음성 문장 생성 — 긴박도별 구조 (매우 가까이: 행동 먼저 / 가까이: 행동 나중 / 보통 이하: 정보만)
- [x] TTS 음성 출력 (gTTS + pygame)
- [x] 카메라 방향 회전 보정 — 시계 산술(+N시간)로 절대 방향 환산

### 서버 / API
- [x] FastAPI `/detect` 엔드포인트 (이미지 POST → 문장 반환)
- [x] 공간 스냅샷 DB (SQLite) — 장소별 변화 감지
- [x] Gradio 데모 UI (localhost:7860)
- [x] `--share` 플래그로 실기기 공개 URL 생성

### 미완성 (다음 단계)
- [ ] Depth Anything V2 거리 추정 활성화 (`depth_anything_v2_vits.pth` 필요)
- [ ] STT 키워드 인식 (Google Speech API, 마이크 입력)
- [ ] Android 앱 — 카메라 캡처 → HTTP POST → TTS 재생
- [ ] Android 카메라 방향 자동 감지 (나침반 + 가속도계 → `camera_orientation` API 전달)

## 방향 구역

| 이미지 x 비율 | 시계 방향 | 위험도 가중치 |
|-------------|---------|------------|
| 0.00 ~ 0.11 | 8시 | 0.3 |
| 0.11 ~ 0.22 | 9시 | 0.5 |
| 0.22 ~ 0.33 | 10시 | 0.7 |
| 0.33 ~ 0.44 | 11시 | 0.9 |
| 0.44 ~ 0.56 | 12시 | 1.0 |
| 0.56 ~ 0.67 | 1시 | 0.9 |
| 0.67 ~ 0.78 | 2시 | 0.7 |
| 0.78 ~ 0.89 | 3시 | 0.5 |
| 0.89 ~ 1.00 | 4시 | 0.3 |

---

## 거리 구역

| bbox 면적 비율 | 대략 거리 | 레이블 | 위험도 가중치 |
|-------------|---------|--------|------------|
| > 0.25 | ~0.5m | 매우 가까이 | 1.0 |
| > 0.12 | ~1.0m | 가까이 | 0.8 |
| > 0.04 | ~2.0m | 보통 | 0.5 |
| > 0.01 | ~4.0m | 멀리 | 0.2 |
| ≤ 0.01 | ~4.0m+ | 매우 멀리 | 0.1 |

## 실행 방법

```bash
# 로컬 실행
conda activate voiceguide
python app.py

# 실기기 테스트 (공개 URL 생성)
python app.py --share
# → 터미널에 출력된 https://xxxx.gradio.live 를 실기기 브라우저에서 열면 됨
```
