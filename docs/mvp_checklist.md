# VoiceGuide MVP 정의서 및 체크리스트

## MVP 목표

시각장애인이 스마트폰 카메라로 주변을 찍으면, AI가 장애물의 **위치·방향·거리·행동 안내**를 음성으로 실시간 안내한다.

---

## 데모 환경

| 항목 | 내용 |
|------|------|
| **실기기** | Samsung Galaxy Z Fold 4 (SM-F936N) |
| **에뮬레이터** | 사용 안 함 (실기기 데모) |
| **서버** | PC (Intel Core Ultra 7 265K, RTX 5060) |
| **연결 방식** | 같은 Wi-Fi 내 로컬 IP (`http://192.168.x.x:8000`) |
| **온디바이스 모드** | 서버 없이 폰 단독 동작 (yolo11m.onnx 탑재) |
| **데모 시나리오** | 폰 들고 장애물 앞 이동 → 1초마다 자동 분석 → 음성 안내 |

## 베이스 모델 실험 / 탑재 / 추론 결과

### 사용 모델

| 모델 | 파라미터 | 크기 | mAP50 | 비고 |
|------|---------|------|-------|------|
| YOLO11n (초기) | 2.6M | 5.4MB | 39.5 | 속도 빠름, 정확도 낮음, 오탐 많음 |
| YOLO11m (중간) | 20.1M | 38.8MB | 51.5 | 정확도/속도 균형 |
| **yolo11m_indoor.pt (최종)** | 20.1M | 39MB | 51.5+계단 | YOLO11m 파인튜닝, 계단(stairs) 클래스 추가 |

### 파인튜닝 결과

- **학습 데이터**: 계단 이미지 404장 (DuckDuckGo 수집 + YOLO pseudo-label)
- **에포크**: 60 (워밍업 10 + 메인 50)
- **GPU**: RTX 5060, 약 9분 소요
- **계단 클래스 성능**: Precision 0.917 / Recall 1.000 / **mAP50 = 0.992**

### 추론 환경

- **플랫폼**: PC GPU (RTX 5060, CUDA)
- **프레임워크**: Ultralytics 8.3.2 + PyTorch 2.x
- **입력 해상도**: 640×640
- **신뢰도 임계값**: 0.60 (기본) / 소형 물체 0.70~0.72 별도 적용
- **추론 방식**: Android 앱 → 이미지 전송 → FastAPI 서버 추론 → 결과 반환
- **온디바이스 방식**: yolo11m.onnx (76.8MB) → ONNX Runtime Android

## 기능 체크리스트

### 핵심 파이프라인

- [x] YOLO11m 장애물 탐지 (COCO 80클래스 + 계단)
- [x] **YOLO11m 파인튜닝** — 계단(stairs) 클래스 추가 학습 (mAP50 0.992)
- [x] 방향 판단 → 이미지 9구역, 시계 방향(8시~4시) → 자연어 변환
- [x] **Depth Anything V2 거리 추정** — GPU 실시간 추론, depth_source="v2"
- [x] 거리 수치 출력 → 1m 미만은 cm, 1m 이상은 m (안전 우선 보수 추정)
- [x] **깊이 맵 기반 계단/낙차/턱 감지** — YOLO 보완, 바닥 12구역 분석
- [x] 위험도 계산 → 방향·거리·바닥여부 가중치 곱, 내림차순 정렬
- [x] **긴박도 4단계 문장 생성** — 0.5m/1m/2.5m 기준 분기
- [x] 이/가 조사 자동화 → 받침 유무 판단
- [x] TTS 음성 출력 → gTTS + 파일 캐시 (반복 문장 즉시 재생)

### 서버 / API

- [x] FastAPI `/detect` 엔드포인트 (이미지 POST → 문장 반환)
- [x] `camera_orientation` 파라미터 지원 (front/back/left/right)
- [x] **`/stt` 엔드포인트** — PC 마이크 음성 인식
- [x] **객체 추적기 (EMA)** — 프레임 간 거리 평균화, 접근/소멸 감지
- [x] 공간 스냅샷 DB (SQLite) → 장소별 변화 감지 (등장/소멸)
- [x] 서버 시작 시 YOLO 워밍업 (lifespan 방식)
- [x] **전역 예외 핸들러** — 오류 시 "분석 중 오류" 음성 문장 반환
- [x] Gradio 데모 UI — 장애물/찾기/확인 모드, 바운딩 박스 시각화

### Android 앱

- [x] CameraX 라이브 프리뷰 (실시간 카메라 화면)
- [x] **1초마다** 자동 캡처 → 서버 전송 → 결과 수신
- [x] 분석 시작 즉시 첫 캡처 (대기 없음)
- [x] Android 기본 TTS 한국어 음성 출력 (속도 1.1배)
- [x] 같은 문장 반복 안내 방지
- [x] **STT 음성 명령** — "주변 알려줘/찾아줘/이거 뭐야" → 장애물/찾기/확인 모드
- [x] **카메라 방향 자동 감지** — 가속도 센서 → front/left/right/back
- [x] **WiFi SSID 자동 수집** — 공간 기억 연동
- [x] **온디바이스 ONNX 추론** — yolo11m.onnx, 서버 없이 폰 단독 동작
- [x] **Failsafe 음성 경고** — 3회 연속 실패 시 "서버 연결 끊겼어요. 주의해서 이동하세요."
- [x] **Watchdog** — 6초 무응답 시 "분석이 중단됐어요. 주의해서 이동하세요."
- [x] 서버 실패 시 온디바이스 모드 자동 전환

## 방향 구역

| 이미지 x 비율 | 시계 방향 | 자연어 표현 | 위험도 가중치 |
|-------------|---------|-----------|------------|
| 0.00 ~ 0.11 | 8시 | 왼쪽 | 0.3 |
| 0.11 ~ 0.33 | 9~10시 | 왼쪽 / 왼쪽 앞 | 0.5 ~ 0.7 |
| 0.33 ~ 0.56 | 11~12시 | 왼쪽 앞 / 바로 앞 | 0.9 ~ 1.0 |
| 0.56 ~ 0.78 | 1~2시 | 오른쪽 앞 | 0.9 ~ 0.7 |
| 0.78 ~ 1.00 | 3~4시 | 오른쪽 | 0.5 ~ 0.3 |

## 거리 추정 방식

### 1차: Depth Anything V2 (ViT-S) — 기본

| 깊이값 범위 | 레이블 | 위험도 가중치 |
|-----------|--------|------------|
| ≤ 0.8m | 매우 가까이 | 1.0 |
| ≤ 2.0m | 가까이 | 0.8 |
| ≤ 4.0m | 보통 | 0.5 |
| ≤ 7.0m | 멀리 | 0.2 |
| > 7.0m | 매우 멀리 | 0.1 |

### 2차: bbox 면적 비율 — Depth V2 모델 없을 때 자동 fallback

| bbox 면적 비율 | 대략 거리 | 레이블 |
|-------------|---------|--------|
| > 0.25 | ~0.5m | 매우 가까이 |
| > 0.12 | ~1.0m | 가까이 |
| > 0.04 | ~2.0m | 보통 |
| > 0.01 | ~4.0m | 멀리 |
| ≤ 0.01 | ~4.0m+ | 매우 멀리 |

## 실행 방법

```bash
# 환경 활성화
conda activate ai_env

# 1회 설정 (최초 1회만)
pip install -r requirements.txt
python patch_gradio_client.py
python -c "import urllib.request; urllib.request.urlretrieve(
    'https://huggingface.co/depth-anything/Depth-Anything-V2-Small/resolve/main/depth_anything_v2_vits.pth',
    'depth_anything_v2_vits.pth'); print('모델 다운로드 완료')"

# 서버 실행
cd VoiceGuide
uvicorn src.api.main:app --host 0.0.0.0 --port 8000

# Gradio 데모
python app.py           # 로컬
python app.py --share   # 폰에서 접속 가능 URL 생성

# 로컬 IP 확인 (Android 앱 연결용)
ipconfig
```
