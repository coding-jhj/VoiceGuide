# 서버가 왜 필요한가?

> 강사님이 서버를 만들라고 하셔서 — 이유 정리

---

## 한 줄 요약

**폰 혼자서는 못 하는 무거운 AI 연산을 PC가 대신 해주기 위해서.**

---

## 서버 없을 때 vs 있을 때

| | 서버 없음 (온디바이스) | 서버 있음 |
|--|--|--|
| **YOLO 탐지** | 폰에서 직접 (ONNX) | PC에서 실행 |
| **거리 추정** | bbox 면적으로 근사 (오차 큼) | Depth Anything V2로 정밀 추정 |
| **계단 감지** | YOLO만 | YOLO + Depth 12구역 이중 감지 |
| **공간 기억** | SharedPreferences (단순 저장) | SQLite DB (재방문 변화 감지) |
| **처리 속도** | 느림 (폰 CPU) | 빠름 (PC GPU) |
| **인터넷** | 불필요 | WiFi 필요 |

---

## 서버가 구체적으로 하는 일

### Depth Anything V2 — 핵심

거리를 정밀하게 추정하려면 94MB짜리 딥러닝 모델이 필요한데,
갤럭시 Z Fold 4에서 돌리면 1초 이상 걸려요.
PC GPU(RTX 5060)에서 돌리면 100ms 안에 끝나요.

### 계단 감지 이중화

Depth 맵을 12구역으로 분석해서 YOLO가 못 잡는 낙차·턱도 추가로 감지해요.
이 연산도 무거워서 서버에서 처리하는 게 맞아요.

### 공간 기억

재방문 시 "의자가 생겼어요" 같은 걸 말하려면
이전 방문 데이터를 어딘가에 저장해야 하는데,
서버의 SQLite DB가 그 역할을 해요.
폰만으로는 앱을 지우면 기억이 사라지고, 여러 기기에서 공유도 안 돼요.

---

## 서버가 있어도 폰 단독 동작이 되는 이유

서버가 꺼지거나 WiFi가 끊겨도 안내가 멈추면 안 되니까
폰에 ONNX 모델을 내장해서 fallback으로 동작하게 만들었어요.

```
서버 연결 성공  →  Depth V2 + 정밀 거리 + 공간 기억  (풀 기능)
서버 연결 실패  →  폰 ONNX 단독 추론              (기본 기능)
```

---

## 강사님이 만들라고 한 이유

AI 서비스 개발에서 **엣지(폰) + 서버 분리 구조**가 실무 표준이에요.

- 무거운 모델은 서버에서
- 가벼운 것만 디바이스에서
- 서버 죽으면 디바이스 fallback

이 구조를 직접 구현해봤다는 게 포트폴리오로도 의미 있고,
강사님 입장에서 "AI 서버 개발" 역량을 확인하는 부분이에요.
