## 정환주 — Android + 서버 통신

**핵심**: 앱에서 찍어서 보내고, 받아서 소리 내는 것

| 업무 | 설명 |
|---|---|
| Android 개발환경 세팅 | Android Studio + 실기기 연결 |
| 카메라 이미지 캡처 | 버튼 → 사진 찍기 |
| WiFi SSID 읽기 | `WifiManager`로 현재 WiFi 이름 읽기 |
| HTTP POST 전송 | 이미지 + SSID → 서버로 전송 (OkHttp/Retrofit) |
| JSON 응답 파싱 | objects, changes 추출 |
| Android TTS 재생 | 음성 결과 재생 |
| 버튼 fallback | STT 불안정 시 버튼 입력 모드 |
| 시나리오 3개 앱에서 작동 확인 | end-to-end 테스트 |

**난이도**: ★★★☆☆

## 신유득 — FastAPI 서버 + 공간 기억

**핵심**: 이미지 받고, 비전·음성 함수 연결하고, 공간 기록 저장하는 허브

| 업무 | 설명 |
|---|---|
| FastAPI 프로젝트 구조 설계 | 라우터, 미들웨어 세팅 |
| `POST /detect` API 구현 | 이미지 수신 → 비전 함수 호출 → 응답 반환 |
| SQLite DB 설계 + 연결 | 공간 스냅샷 저장 스키마 |
| `POST /spaces/snapshot` API | 탐지 결과 저장 |
| 변화 감지 로직 | 이전 기록 vs 현재 비교 → changes 생성 |
| ngrok 외부 접근 설정 | 앱에서 접근 가능하도록 |
| 서버 안정화 + 에러 핸들링 | 타임아웃, 예외 처리 |
| API 명세 문서 작성 | 팀 전체가 볼 수 있게 요청/응답 형식 정리 |

**난이도**: ★★★★☆

## 김재현 — YOLO + 방향/위험도

**핵심**: 이미지에서 물체 찾고, 방향 판단하고, 위험도 계산하는 함수 만들기

| 업무 | 설명 |
|---|---|
| YOLO11n 설치 + 5종 탐지 확인 | person / chair / table / backpack / cell phone |
| 방향 판단 로직 | bbox 중심 x좌표 → left / center / right |
| 위험도 스코어 계산 | 방향 + 거리 조합 → 상위 1~2개 선택 |
| `detect()` 함수 완성 | 백엔드가 호출할 단일 함수 |
| 인식률 테스트 + 기록 | 조명/각도/거리별 탐지 성공률 정리 |
| 오인식 케이스 분석 | 잘 안 되는 케이스 수집 + 개선 |
| 테스트 이미지 수집 | 실내 5종 물체 각 30장 이상 직접 촬영 |
| 시나리오별 탐지 결과 검증 | 장애물 안내 / 물건 찾기 / 물건 확인 |

**난이도**: ★★★★☆

## 문수찬 — Depth Anything V2 + STT/TTS

**핵심**: 거리 측정 모델 붙이고, 음성 입출력 만들기

| 업무 | 설명 |
|---|---|
| Depth Anything V2 서버 탑재 | Small 모델 로드, 서버에서 실행 |
| bbox 중심 깊이값 추출 | `depth_map[cy][cx]` → 가까이/보통/멀리 분류 |
| depth 임계값 튜닝 | 실내 환경 기준으로 실험 조정 |
| `detect_and_depth()` 완성 | 김재현의 detect + depth 통합 함수 |
| gTTS 설치 + 음성 출력 확인 | 한국어 TTS 기본 작동 |
| STT 연결 (Google Speech API) | 한국어 음성 → 텍스트 |
| 키워드 매칭 | "앞에 뭐 있어" / "핸드폰 찾아줘" / "이거 뭐야" |
| 소음 환경 테스트 | 카페/지하철 수준 소음 인식률 기록 |

**난이도**: ★★★★☆

## 임명광 — 문장 템플릿 + 기획/발표

**핵심**: 탐지 결과를 자연스러운 한국어로 바꾸고, 발표 자료 만들기

| 업무 | 설명 |
|---|---|
| 방향+거리 조합 문장 템플릿 | 30~50개 케이스 작성 (예: "왼쪽 바로 앞에 의자가 있어요") |
| 변화 감지 → 안내 문장 | "가방이 하나 더 있어요" 자연어화 |
| `build_sentence()` 함수 완성 | 백엔드가 호출할 문장 생성 함수 |
| 시나리오 테스트 케이스 작성 | 입력/기대출력 기준 문서 (통합 테스트용) |
| 시각장애인 유튜브 영상 3~5개 분석 | 실제 불편 정리 → 발표 근거 자료 |
| 기존 서비스 비교표 작성 | Google Lookout / Seeing AI / Be My Eyes |
| PPT 작성 + 발표 대본 | 문제 → 솔루션 → 기술 → 결과 → 기대효과 |
| 데모 시나리오 스크립트 | 발표용 데모 3가지 흐름 대본 |

**난이도**: ★★★☆☆

## 팀 간 연결 포인트

### 온디바이스 흐름 (기본 — 서버 불필요)

```
정환주 (Android 앱)
  → YoloDetector.kt (ONNX 추론)      ← 김재현 모델 파인튜닝 결과
  → SentenceBuilder.kt (문장 생성)   ← 임명광 문장 설계 반영
  → Android TTS 재생
  → SharedPreferences (장소 저장)    ← 신유득 DB 설계 아이디어
```

### 서버 연동 흐름 (선택 — 정확도 향상)

```
정환주 (Android)
  ↓ POST /detect {image, wifi_ssid, mode, query_text}
신유득 (FastAPI 서버)
  → detect_and_depth()  ← 김재현 + 문수찬 구현
  → build_sentence()    ← 임명광 구현
  → DB 저장 + 변화 감지
  ↓ {sentence, objects, hazards, changes}
정환주 (Android) → TTS 재생
```
