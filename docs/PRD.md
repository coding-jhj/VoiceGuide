---

## 1. 문제 정의

### 배경

시각장애인은 실내 이동 시 주변 환경을 실시간으로 파악하기 어렵습니다.
기존 AI 서비스(Google Lookout, Seeing AI)는 물체를 **설명**하지만
**행동을 안내하지 않습니다.**

| 서비스 | 출력 예시 | 한계 |
|--------|---------|------|
| Google Lookout | "의자가 있습니다" | 방향·거리 없음 |
| Microsoft Seeing AI | "왼쪽에 의자가 있습니다" | 행동 안내 없음 |
| **VoiceGuide** | "왼쪽 바로 앞에 의자가 있어요. 오른쪽으로 비켜보세요." | — |

### 최종 문제 정의 (한 문장)

> **"시각장애인은 실내 이동 중 장애물의 위치와 회피 방향을 즉각적으로 안내받지 못하고 있으므로, 카메라와 음성 명령을 결합한 행동 중심 안내 서비스가 필요하다."**

## 2. 타겟 사용자

### 페르소나: 김지수 (34세, 선천성 시각장애)

```
상황    : 혼자 집·직장·카페 등 실내 공간 이용
목표    : 타인 도움 없이 실내에서 안전하게 이동
Pain    : 익숙하지 않은 공간에서 바닥 가방·의자에 부딪힘
기기    : 스마트폰 보유 (스크린리더 사용 중)
요구사항 : 빠르고 간결한 안내 / 소음 환경에서도 작동 / 일관된 UX
```

### 사용자 니즈 근거 (논문 기반)

- **UC Davis 사용자 조사(2024, arxiv:2504.06379)**: 시각장애인이 가장 원하는 것은 "행동 유도 안내"
- **Nature Scientific Reports(2025)**: 기존 기기 실사용 포기 원인 1위는 "복잡한 UX"
- **Zagreb 대학(2024)**: 75% 이상이 "비용 문제"를 가장 큰 장벽으로 꼽음
  → VoiceGuide는 스마트폰 하나로 해결, 추가 비용 없음

## 3. 핵심 기능 (구현 완료)

| 모드 | 기능 | 트리거 키워드 예시 | 출력 예시 |
|------|------|----------------|---------|
| `장애물` | 장애물 안내 | "앞에 뭐 있어", "주변 알려줘", "분석해줘" | "오른쪽 앞에 의자가 있어요. 약 1.5m. 왼쪽으로 피해가세요." |
| `찾기` | 특정 물건 찾기 | "의자 찾아줘", "가방 어디있어" | "의자는 왼쪽 앞에 있어요. 약 2m." |
| `확인` | 물건 확인 | "이거 뭐야", "이게 뭔지" | "오른쪽에 노트북도 있어요. 약 1m." |
| `저장` ⭐ | 현재 위치 저장 | "여기 저장해줘 편의점", "기억해줘 화장실" | "편의점을 저장했어요." |
| `위치목록` ⭐ | 저장 장소 확인 | "저장된 곳 알려줘", "내 장소 목록" | "저장된 장소는 편의점, 화장실, 강의실이에요." |

> ⭐ 개인 네비게이팅: WiFi SSID 기반 장소 기억. 앱 단독 동작 (서버 불필요).

### MVP 완성 기준 (단 하나)

```
카메라로 의자 찍기
    → YOLO 탐지
    → "의자가 왼쪽에 있습니다" 음성 출력
이 흐름 하나가 작동하면 MVP 완성.
```

## 4. 인식 대상 (COCO 80클래스 전체 + 계단 파인튜닝 = 81클래스)

COCO 80클래스를 전부 활용하고 계단을 직접 파인튜닝으로 추가했습니다.  
`YOLO_WORLD=1` 환경변수로 전동킥보드·볼라드 등 한국 특화 클래스도 추가 탐지 가능합니다.

| 환경 | 카테고리 | 주요 클래스 | 위험도 배수 |
|------|---------|-----------|-----------|
| **야외** | 이동 차량 | 자동차·오토바이·버스·트럭·기차·비행기·보트 | **3.0~4.0×** |
| **야외** | 이동 수단 | 자전거·스케이트보드 | 2.0× |
| **야외** | 교통 시설 | 신호등·소화전·정지 표지판·주차 미터기 | 0.6~1.2× |
| **야외** | 동물 | 개·고양이·말·소·새·양·코끼리·곰·얼룩말·기린 | 1.2~4.0× |
| **실내외** | 날카로운 물체 | 칼·가위·유리잔·야구 방망이 | **1.5~2.5×** |
| **실내외** | 대형 가구·구조물 | 의자·소파·테이블·침대·냉장고·벤치 | 1.0× |
| **실내외** | 바닥 장애물 | 배낭·핸드백·여행가방·우산·공·원반 | 1.0~1.4× |
| **실내외** | 미끄럼 위험 | 바나나 등 음식류 (바닥 감지) | 1.0× |
| **실내외** | 확인용 | 휴대폰·노트북·병·컵·책·시계 | 0.5~1.0× |

## 5. 플랫폼 전략

**Android 앱이 유일한 실사용 플랫폼입니다.** 실제 시각장애인은 서버를 설치하거나 Gradio 웹을 쓸 수 없습니다.

| 플랫폼 | 용도 | 상태 |
|--------|------|------|
| **Android 앱** | 실사용 (주) | ✅ 완성 — 서버 없이 단독 동작 |
| Python + Gradio | 발표·시연 데모 | ✅ 완성 — 강사님 시연용 |
| FastAPI 서버 | 서버 연동 시 정확도 향상 | ✅ 완성 — 선택 사항 |

**Android 앱 동작 방식:**
- 기본: 폰에서 ONNX 추론 (서버 불필요, 완전 오프라인)
- 선택: 서버 URL 입력 시 Depth Anything V2로 거리 정확도 향상

## 6. 성공 지표

### 기술 지표

| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 실내 장애물 탐지 | 100% | 가구·바닥 장애물·계단 — bbox + Depth 이중 감지 |
| 야외 차량 탐지 | 100% | 자동차·버스·오토바이 conf 0.40, 위험도 3~4배 강화 |
| 야외 동물 탐지 | 100% | 개·말 — 돌발 행동 위험 1.8~2.5배 |
| 음성 응답 시간 | 3초 이내 | 이미지 입력 → 음성 출력 측정 |
| STT 인식률 | 100% | 5모드 × 키워드 15개+ / 미매칭 시 장애물 모드 fallback |
| 방향 판단 정확도 | 100% | 8시~4시 9구역 결정론적 분류 |
| 개인 네비게이팅 | 100% | WiFi SSID 기반 장소 저장·찾기·목록 |

### 발표 기준

| 기준 | 내용 |
|------|------|
| 데모 시나리오 | 3가지 중 최소 1가지 실시간 작동 |
| 실패 대비 | 데모 영상 사전 녹화 필수 |

## 7. 차별점

```
기존 서비스 : 환경을 설명한다
VoiceGuide : 환경을 기억하고, 행동을 안내하며, 장소를 기억한다
```

| 차별점 | 구현 방식 | 경쟁 서비스 |
|--------|---------|-----------|
| 행동 중심 안내 | 방향 + 거리 + 회피 방향 동시 제공 | Google Lookout: 방향 없음 |
| 계단·낙차 감지 | Depth map 12구역 분석 → 바닥 위험 최우선 | Seeing AI: 계단 감지 없음 |
| 개인 네비게이팅 ⭐ | 장소 저장/찾기 (WiFi 기반) | 없음 |
| 완전 오프라인 | 서버 없이 폰 단독 ONNX 추론 | 대부분 클라우드 의존 |
| 공간 기억 | 재방문 시 변화만 안내 | 매번 동일 설명 반복 |
| 한국어 특화 | 조사 자동화, 자연스러운 구어체 | 영어 위주 |
