## 1. 문제 정의

### 배경

시각장애인은 실내 이동 시 주변 환경을 실시간으로 파악하기 어렵습니다.
기존 AI 서비스(Google Lookout, Seeing AI)는 물체를 **설명**하지만
**행동을 안내하지 않습니다.**

| 서비스 | 출력 예시 | 한계 |
|--------|---------|------|
| Google Lookout | "의자가 있습니다" | 방향·거리 없음 |
| Microsoft Seeing AI | "왼쪽에 의자가 있습니다" | 행동 안내 없음 |
| **VoiceGuide** | "왼쪽 바로 앞에 의자가 있어요. 오른쪽으로 비켜보세요." | — |

### 최종 문제 정의 (한 문장)

> **"시각장애인은 실내 이동 중 장애물의 위치와 회피 방향을 즉각적으로 안내받지 못하고 있으므로, 카메라와 음성 명령을 결합한 행동 중심 안내 서비스가 필요하다."**

## 2. 타겟 사용자

### 페르소나: 김지수 (34세, 선천성 시각장애)

```
상황    : 혼자 집·직장·카페 등 실내 공간 이용
목표    : 타인 도움 없이 실내에서 안전하게 이동
Pain    : 익숙하지 않은 공간에서 바닥 가방·의자에 부딪힘
기기    : 스마트폰 보유 (스크린리더 사용 중)
요구사항 : 빠르고 간결한 안내 / 소음 환경에서도 작동 / 일관된 UX
```

### 사용자 니즈 근거 (논문 기반)

- **UC Davis 사용자 조사(2024, arxiv:2504.06379)**: 시각장애인이 가장 원하는 것은 "행동 유도 안내"
- **Nature Scientific Reports(2025)**: 기존 기기 실사용 포기 원인 1위는 "복잡한 UX"
- **Zagreb 대학(2024)**: 75% 이상이 "비용 문제"를 가장 큰 장벽으로 꼽음
  → VoiceGuide는 스마트폰 하나로 해결, 추가 비용 없음

## 3. 핵심 기능 (우선순위 고정)

> **AI 도구 참고**: 아래 3가지 기능만 구현합니다.
> 범위를 벗어나는 기능 추가는 팀 합의 없이 불가합니다.

| 우선순위 | 기능 | 트리거 키워드 | 출력 예시 |
|---------|------|------------|---------|
| **1** (필수) | 장애물 안내 | "앞에 뭐 있어", "주변 알려줘" | "정면 가까이에 의자가 있습니다. 멈추세요." |
| **2** (필수) | 물건 찾기 | "핸드폰 찾아줘", "가방 어딨어" | "핸드폰은 오른쪽에 있습니다." |
| **3** (선택) | 물건 확인 | "이거 뭐야", "이게 뭐야" | "카메라 중앙의 물건은 가방입니다." |

### MVP 완성 기준 (단 하나)

```
카메라로 의자 찍기
    → YOLO 탐지
    → "의자가 왼쪽에 있습니다" 음성 출력
이 흐름 하나가 작동하면 MVP 완성.
```

## 4. 인식 대상 (5종 고정)

> **AI 도구 참고**: 아래 5종만 처리합니다.
> 다른 클래스가 탐지되어도 무시하거나 "알 수 없는 물체"로 처리하세요.

| 한국어 | YOLO class명 | 선정 이유 |
|--------|------------|---------|
| 사람 | `person` | 실내 가장 흔한 장애물 |
| 의자 | `chair` | 충돌 위험 1순위 |
| 테이블 | `dining table` | 충돌 위험 2순위 |
| 가방 | `backpack`, `suitcase` | 바닥 장애물 대표 |
| 휴대폰 | `cell phone` | 물건 찾기 대표 사물 |

```python
# 코드에서 사용할 상수
TARGET_CLASSES = {
    "person":       "사람",
    "chair":        "의자",
    "dining table": "테이블",
    "backpack":     "가방",
    "suitcase":     "가방",
    "cell phone":   "휴대폰",
}
```

## 6. 성공 지표

### 기술 지표

| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 5종 탐지 성공률 | 80% 이상 | 실내 표준 조명, 각 물체 30장 테스트 |
| 음성 응답 시간 | 3초 이내 | 이미지 입력 → 음성 출력 측정 |
| STT 인식률 (조용한 환경) | 90% 이상 | 키워드 3종 × 10회 테스트 |
| STT 인식률 (소음 환경) | 70% 이상 | 카페 수준 소음 기준 |
| 방향 판단 정확도 | 90% 이상 | left/center/right 정답 비교 |

### 발표 기준

| 기준 | 내용 |
|------|------|
| 데모 시나리오 | 3가지 중 최소 1가지 실시간 작동 |
| 실패 대비 | 데모 영상 사전 녹화 필수 |

## 7. 차별점

```
기존 서비스 : 환경을 설명한다
VoiceGuide : 환경을 기억하고 행동을 안내한다
```

| 차별점 | 구현 방식 |
|--------|---------|
| 행동 중심 안내 | 방향 + 거리 + 회피 방향 동시 제공 |
| 공간 기억 | WiFi SSID 기반 공간 식별 → 재방문 시 변화만 안내 |
| 접근성 | 스마트폰 하나로 동작, 추가 비용 없음 |
| 기술 타이밍 | YOLO11n + Depth Anything V2 조합이 2024년 이후 처음으로 현실적 선택지 |
