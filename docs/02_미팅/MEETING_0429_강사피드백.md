# 4/29 강사님 미팅 피드백 및 즉일 반영 내용

## 핵심 지시사항 (우선순위 순)

### 🔴 1순위 — 추론 속도 / FPS

- **목표: FPS 10 이상** (미달 시 경량화 또는 로직 변경 필요)
- Depth Anything + 바운딩박스 + 음성이 섞여서 느린 게 주원인
- 거리 추정은 후순위로 미루거나 다른 방법 검토
- 추론 모델 → Depth 계산 → 음성 안내 파이프라인 재검토

**즉일 반영:**
- NNAPI 하드웨어 가속 추가 (`YoloDetector.kt`)
- CPU 스레드 4개 병렬 처리 추가
- 캡처 간격 800ms (기존 1000ms)
- FPS 스파크라인 그래프 UI 표시: `2.1fps ▄▃▅▄▃▅ | 180ms`

---

### 🔴 2순위 — 서버 GCP 마이그레이션

**스펙 지정:**
- GPU 없음 (CPU 전용)
- 디스크 30GB 이하
- 리전: 가장 가까운 곳 → `asia-northeast3` (서울)
- OS: Ubuntu 22.04 LTS 또는 24.04 LTS
- VS Code로 SSH 붙여서 개발

**절차:**
1. GCP Compute Engine 인스턴스 생성 (상세: `docs/GCP_SERVER_SETUP.md`)
2. IP 나오면 VS Code SSH 연결
3. 서버 코드 올리고 uvicorn 실행
4. Android 앱 서버 URL 입력창에 IP 입력
5. LTE 환경에서 외부 접속 확인

**현재 상황:** 신유득 로컬에서만 동작 중, Supabase 연동은 됨

---

### 🟡 3순위 — 음성명령 딜레이

- **문제: 명령 후 3초 딜레이** → 수정 필요
- STT 인식 완료 판단 시간이 너무 길었음

**즉일 반영:**
- 침묵 감지 시간: 1200ms → **700ms** (말 끝나면 0.7초 후 인식 완료)
- 부분 인식 결과 실시간 화면 표시 (말하는 중에도 인식 확인 가능)

---

### 🟡 4순위 — 로그 / 디버깅

- **Logcat 로그 필수** — 안 되는 부분 원인 파악용
- 로그를 그래프로 찍어서 잘 보이게
- 필요한 로그만 뽑아서 분석

**즉일 반영:**
- 구조화 성능 로그 추가: `tag:VG_PERF`
  ```
  decode|12|infer|180|dedup|3|total|195|objs|2
  mode|server|server_ms|243|net_ms|89|total|332
  ```
- Logcat 필터: `tag:VG_PERF` 또는 `tag:VG_DETECT`

---

## 로그 분석 방법

### Android Studio Logcat에서 필터

```
tag:VG_PERF    ← 성능 측정 (FPS, 각 단계별 ms)
tag:VG_DETECT  ← 탐지 결과 (물체 수, 생성 문장)
```

### 성능 로그 형식

```
VG_PERF: decode|12|infer|180|dedup|3|total|195|objs|2
         ↑디코딩  ↑YOLO추론  ↑중복제거 ↑총합  ↑탐지수
```

### FPS 목표

| FPS | 판단 | 조치 |
|-----|------|------|
| 10+ | 정상 | 유지 |
| 5~10 | 경계 | 모델 경량화 검토 |
| 5 미만 | 느림 | Depth 비활성화, YOLO nano 전환 |
