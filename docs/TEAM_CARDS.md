# VoiceGuide 팀원별 핵심 카드

> 발표 전 자기 카드만 읽어도 충분합니다.
> 다른 팀원 모듈은 "무엇을 받아서 무엇을 내보내는지"만 알면 됩니다.

---

## 전체 흐름 한 줄 요약

```
폰 카메라 → 사진 → 서버(YOLO+Depth) → 위험도 계산 → 한국어 문장 → 폰 TTS
```

---

## 정환주 (조장 · Android 앱)

### 내가 만든 것
Android 앱 전체. 카메라·STT·TTS·센서를 연결하고, 서버 없이도 동작하게 만들었다.

### 핵심 흐름 (외울 것)
```
앱 시작
  → "음성 안내를 시작할까요?" (음성 자동 시작)
  → "네" 말하면 카메라 ON
  → 1초마다 사진 촬영
  → ONNX 모델(온디바이스) 또는 서버로 전송
  → 결과 문장을 TTS로 읽어줌
  → 동시에 가속도·조도 센서 감시
```

### 내가 설명할 수 있어야 하는 것

**Q. 서버 없이도 동작하는 이유?**
ONNX 형식으로 변환한 YOLO 모델이 앱 안에 내장되어 있어서, 서버 연결 없이 폰 CPU/GPU로 직접 추론합니다. 서버가 연결되면 Depth V2 정밀 거리 추정이 추가됩니다.

**Q. 음성 명령은 어떻게 분류해요?**
Android SpeechRecognizer로 텍스트 변환 후, VoiceGuideConstants.kt의 STT_KEYWORDS 맵에서 키워드 포함 여부를 순서대로 검색합니다. 아무것도 안 걸리면 기본 장애물 모드로 fallback합니다.

**Q. 낙상 감지는 어떻게 해요?**
가속도 센서로 크기(magnitude)를 계산합니다. 자유낙하(~0)가 감지된 직후 충격(25+ m/s²)이 오면 낙상으로 판단, 10초 안에 "괜찮아" 응답 없으면 SOS 자동 발송합니다.

### 내 파일
`android/.../MainActivity.kt`, `YoloDetector.kt`, `SentenceBuilder.kt`, `VoiceGuideConstants.kt`

## 신유득 (FastAPI 서버)

### 내가 만든 것
서버의 허브. 각 모듈을 연결하고, DB로 공간 기억을 관리하고, 물체 추적기를 운영한다.

### 핵심 흐름 (외울 것)
```
POST /detect 요청 수신
  → 모드 확인 (저장/위치목록은 이미지 없이 즉시 처리)
  → detect_and_depth() 호출 → YOLO 탐지 + 거리 추정
  → tracker.update() → EMA 거리 평균화
  → db.get_snapshot() → 이전 방문과 비교
  → build_sentence() → 한국어 문장 생성
  → beep/alert_level 결정 → 응답 반환
```

### 내가 설명할 수 있어야 하는 것

**Q. 서버가 왜 필요해요?**
Depth Anything V2 (94MB)를 폰에서 돌리면 1초 이상 걸립니다. PC GPU(RTX 5060)에서 돌리면 100ms 이내입니다. 공간 기억 DB, EasyOCR, GPT Vision도 서버에서 처리합니다.

**Q. 공간 기억이 뭐예요?**
WiFi SSID를 공간 ID로 사용합니다. 같은 WiFi에서 재방문하면 이전 탐지 결과와 비교해서 달라진 것만 안내합니다. "의자가 생겼어요" 같은 식으로요.

**Q. EMA 추적기가 왜 필요해요?**
YOLO가 프레임마다 거리를 다르게 출력합니다 (jitter). 1.0m → 1.3m → 0.9m가 반복되면 TTS가 계속 다른 말을 해서 혼란스럽습니다. EMA(지수이동평균)로 부드럽게 만들어서 안정적인 안내를 합니다.

### 내 파일
`src/api/routes.py`, `src/api/db.py`, `src/api/tracker.py`

## 김재현 (YOLO 탐지)

### 내가 만든 것
물체를 찾고, 방향·거리·위험도를 계산한다. 계단 클래스를 직접 파인튜닝으로 추가했다.

### 핵심 흐름 (외울 것)
```
이미지 입력
  → cv2.flip(img, 1)  ← 좌우 반전 보정 (카메라 mirror 현상)
  → YOLO11m 추론 → 박스(bbox), 클래스, 신뢰도
  → bbox 중심 x좌표 → 9구역 → 시계방향 → "왼쪽 앞"
  → bbox 면적 비율 → 거리 초기 추정
  → 위험도 = 방향가중치 × 거리가중치 × 바닥가중치 × 클래스배수
  → 위험도 상위 3개 반환
```

### 내가 설명할 수 있어야 하는 것

**Q. 왜 YOLO11m이에요? n이 더 빠르지 않아요?**
처음에 YOLO11n을 썼는데 손·팔을 사람으로 오인하는 오탐이 많았습니다. 시각장애인 안전이 걸려 있어서 속도보다 정확도를 우선했고, m으로 바꾸니 오탐이 크게 줄었습니다.

**Q. 계단은 왜 파인튜닝했어요?**
COCO 80클래스에 계단이 없습니다. 이미지 404장을 직접 수집해서 파인튜닝했고, mAP50이 0.992(99.2%)가 나왔습니다.

**Q. 위험도가 어떻게 계산돼요?**
방향가중치(바로 앞=1.0, 왼쪽=0.3) × 거리가중치(코앞=1.0, 멀리=0.1) × 클래스배수(기차=4.0, 의자=1.0)를 곱합니다. 이 값이 높을수록 먼저 안내합니다.

### 내 파일
`src/vision/detect.py`

## 문수찬 (Depth + 음성)

### 내가 만든 것
카메라 하나로 거리를 추정하고, 계단·낙차를 감지한다. PC STT도 담당한다.

### 핵심 흐름 (외울 것)
```
이미지 → Depth Anything V2 → 깊이 맵 (H×W 배열)
  ├─ bbox 영역 하위 30% 깊이값 → 미터 단위 거리 (더 가깝게 추정 = 안전 우선)
  └─ 바닥 12구역 분석
       갑자기 깊어지면 → 낙차/계단 경고
       갑자기 얕아지면 → 턱/계단 경고
       분산 크면      → 울퉁불퉁한 길 경고
```

### 내가 설명할 수 있어야 하는 것

**Q. 거리가 정확해요?**
정확하지 않습니다. Depth V2는 상대적 깊이를 추정하는 모델이라 오차가 있습니다. 그래서 "약 1.2미터" 대신 "가까이", "조금 멀리" 같은 상대 표현을 씁니다. 강사님 지적을 반영한 부분입니다.

**Q. 왜 하위 30%를 쓰나요?**
bbox 안에는 물체 앞면(가까운 부분)과 뒷면(먼 부분)이 섞여 있습니다. 평균을 쓰면 실제보다 멀리 추정됩니다. 안전 우선으로 가장 가까운 쪽 값을 써야 조기 경고가 됩니다.

**Q. YOLO랑 Depth 둘 다 계단 감지하는 이유?**
YOLO는 계단이 시각적으로 보여야 탐지합니다. 작은 낙차나 턱은 안 잡힙니다. Depth는 바닥 깊이 변화를 보는 거라 YOLO가 못 잡는 것도 잡을 수 있습니다.

### 내 파일
`src/depth/depth.py`, `src/depth/hazard.py`, `src/voice/stt.py`

## 임명광 (문장 생성 + 발표)

### 내가 만든 것
탐지 결과를 받아서 시각장애인이 바로 이해할 수 있는 한국어 문장을 만든다.

### 핵심 흐름 (외울 것)
```
물체 정보 입력 (방향, 거리, 클래스, 위험도)
  → 물체 종류 판단 (차량? 동물? 바닥 장애물? 일반?)
  → 거리 판단 (바로 코앞? 가까이? 조금 멀리?)
  → 긴박도에 맞는 문장 구조 선택
  → 한국어 조사 자동화 (의자가/책이, 의자는/책은)
  → 완성된 문장 반환
```

### 내가 설명할 수 있어야 하는 것

**Q. 차량이 왜 특별하게 처리돼요?**
의자는 가만히 있지만 차량은 움직입니다. 의자는 0.5m 기준으로 긴박도를 나누지만, 차량은 8m 이내부터 "조심!"을 외칩니다. 이미 1m까지 기다리면 늦습니다.

**Q. 한국어 조사가 왜 자동화가 필요해요?**
"의자가 있어요"와 "책이 있어요"처럼 받침 유무에 따라 조사가 달라집니다. 글자 코드 계산으로 받침을 판별해서 자동 처리합니다. 없으면 "가방가 있어요" 같은 말이 나옵니다.

**Q. 거리를 수치로 안 말하는 이유?**
"약 1.2미터"처럼 말하면 정확한 것처럼 들리는데 실제로 오차가 큽니다. 강사님 지적 반영해서 "가까이", "조금 멀리"처럼 상대 표현으로 바꿨습니다.

### 내 파일
`src/nlg/sentence.py`, `src/nlg/templates.py`

## 전체 연결 구조 한눈에

```
[정환주: Android]
  카메라 1초 캡처 → 서버 전송 or 온디바이스 추론
  STT 음성 명령 → 22가지 모드 분기
  TTS + 비프음 + 진동으로 출력

        ↕ HTTP (같은 WiFi)

[신유득: 서버 허브]
  /detect → 모드 분기 → 아래 모듈 호출 → 문장 반환

  [김재현: YOLO]          [문수찬: Depth]        [임명광: NLG]
  물체 탐지               거리 추정              문장 생성
  방향·위험도 계산        계단·낙차 감지         조사 자동화
  신호등·색상 분석        울퉁불퉁한 길          긴박도 분기

  [신유득: 추적·DB]
  EMA 거리 평활화
  공간 기억 비교
  EasyOCR·GPT Vision
```
