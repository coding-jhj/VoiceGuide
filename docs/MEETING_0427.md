## 핵심 피드백 3가지

### 1. 경고 피로 — 안내 멘트가 너무 많다

지금은 물체가 보이면 전부 음성으로 말해줘요.
시각장애인 입장에서 1초마다 여러 물체를 다 읽어주면 오히려 혼란스러워요.
강사님 제안: 위험한 건 음성, 덜 중요한 건 비프음으로 짧게.

→ 경고를 계층으로 나눠야 함

| 단계 | 기준 | 출력 |
|------|------|------|
| 즉각 위험 | 차량 8m 이내, 계단 1m, 칼·가위 3m | 음성 (즉시) |
| 일반 장애물 | 1m 이내 물체 | 비프음 |
| 정보성 안내 | 2m 이상 물체, 군중 | 음성 명령 시에만 |

---

### 2. 카메라로 거리 측정은 안 된다

단안 카메라 거리 추정은 오차가 크고 강사님도 그걸 아세요.
"1.2미터" 같은 수치를 말하는 게 과장처럼 보일 수 있어요.

→ 발표할 때는 솔직하게: "Depth V2로 상대 거리 추정, 정밀 측정은 한계가 있어요"
→ 음성 멘트는 수치 대신 상대 표현으로 바꾸기

| 기존 | 변경 |
|------|------|
| "약 1.2미터" | "가까이" |
| "약 2.5미터" | "조금 멀리" |
| "약 0.3미터" | "바로 앞" |

---

### 3. 타겟 유저를 명확히 해라

시각 장애 유형이 여러 가지인데 다 잡으려 하면 아무것도 못 해요.

시각 장애 유형:
- 시력 장애: 사물 식별력 저하
- 시야 장애: 터널 시야, 중심 시야 손실
- 색각 장애: 색맹·색약
- 광각 장애: 눈부심, 야맹증
- 굴절·조절 장애: 초점 문제

→ **전맹(완전 시각 장애)을 주 타겟으로** 설계
→ 다른 유형은 기능 끄기로 대응
→ 발표에서 "전맹 사용자를 주 타겟으로 설계했다"고 명확히 말하기

## 추가하고 싶은 기능

### 신호등 파란불 감지 ⭐

시각장애인분들이 신호등 앞에서 언제 건너야 할지 모르는 게 큰 두려움이에요.
현재 보조 기기는 신호등 앞에서 키를 누르면 불이 바뀔 때 진동으로 알려주는 방식인데,
저희는 카메라로 직접 감지할 수 있어요.

YOLO가 이미 신호등을 탐지하고 있어서,
탐지된 신호등 bbox 영역 안에서 상단(빨간불 위치)과 하단(초록불 위치)의 밝기를 비교하면
색상 판별이 가능해요. 빨간불 → 초록불로 바뀌는 순간을 잡아서
"신호등이 초록불로 바뀌었어요. 건너가세요." 안내.

구현 방향:
- YOLO `traffic light` 탐지 후 bbox crop
- HSV 색공간에서 초록 영역 비율 계산
- 이전 프레임 대비 초록 비율이 올라가면 → 전환 감지

## 강사님 추가 언급

- 바운딩 박스 디버깅이 어렵다 → 로직 설명 준비해야 함

- 버스 번호 인식: OCR보다 Vision AI로 접근

## Seeing AI 기능 — VoiceGuide 반영 계획

Microsoft Seeing AI의 9가지 기능 전체를 VoiceGuide에 흡수하는 방향.

| Seeing AI 기능 | 설명 | VoiceGuide 구현 방향 | 난이도 |
|--------------|------|-------------------|------|
| **Short Text** | 카메라에 보이는 짧은 텍스트 즉시 읽기 | Google ML Kit Text Recognition (온디바이스 OCR) | 🟢 쉬움 |
| **Documents** | 문서·서류 전체를 읽고 음성 출력 | ML Kit Document Scanner + TTS | 🟡 중간 |
| **Handwriting** | 손글씨 인식 후 읽기 | ML Kit Handwriting Recognition | 🟡 중간 |
| **Products** | 바코드 스캔 → 상품명 읽기 | ML Kit Barcode Scanning (ZXing 대체 가능) | 🟢 쉬움 |
| **Currency** | 지폐 종류 인식 → "만원권입니다" | 분류 모델 or GPT Vision API | 🔴 어려움 |
| **People** | 얼굴 인식 → 이름·표정·나이 추정 | ML Kit Face Detection + 등록 기능 | 🔴 어려움 |
| **Scene** | 주변 전체 상황을 문장으로 설명 | 이미 YOLO로 일부 구현, GPT Vision으로 확장 가능 | 🟡 중간 |
| **Color** | 물체·옷 색상을 말해줌 | YOLO bbox 영역 HSV 분석 → 색상명 변환 | 🟢 쉬움 |
| **Light** | 주변 밝기를 소리로 알려줌 | Android 조도 센서 (카메라 불필요) | 🟢 쉬움 |

### 당장 붙일 수 있는 것 (ML Kit 기반, 온디바이스)

- **Short Text** — "텍스트 읽기" 음성 명령 → 카메라 화면 텍스트 즉시 읽기
- **Products** — "이거 뭐야" + 바코드 감지 시 자동으로 상품 정보 읽기
- **Color** — "색깔 알려줘" 명령 → 중앙 물체 색상 말해줌
- **Light** — 조도 센서로 어두운 환경 자동 감지 → "주변이 어두워요"

### 중기 목표

- **Scene** — GPT Vision API로 전체 상황 한 문장 요약 ("카페 내부, 앞에 테이블과 의자가 있어요")
- **Documents** — 문서 촬영 모드 추가, 처방전·메뉴판 등 읽기
- **Handwriting** — 손글씨 메모 읽기

### 보류 (복잡도 대비 효용 낮음)

- **Currency** — 한국 지폐 전용 모델 필요, 별도 학습 필요
- **People** — 얼굴 등록 시스템 필요, 개인정보 이슈

### 음성 명령 모드 확장 (Seeing AI 흡수 후)

| 음성 명령 | 기능 |
|---------|------|
| "주변 알려줘" / "장애물 모드" | 기존 장애물 탐지 |
| "이거 찾아줘 [물체]" | 특정 물체 찾기 |
| "이거 뭐야" | 물체 확인 + 바코드 스캔 |
| "글자 읽어줘" | Short Text OCR |
| "문서 읽어줘" | Document 모드 |
| "색깔 알려줘" | Color 감지 |
| "신호등 봐줘" | 신호등 파란불 감지 |
| "밝기 알려줘" | 조도 센서 |
| "여기 저장해줘 [이름]" | 개인 네비게이팅 |
| "저장된 곳 알려줘" | 장소 목록 |
