## 핵심 피드백 3가지

### 1. 경고 피로 — 안내 멘트가 너무 많다

지금은 물체가 보이면 전부 음성으로 말해줘요.
시각장애인 입장에서 1초마다 여러 물체를 다 읽어주면 오히려 혼란스러워요.
강사님 제안: 위험한 건 음성, 덜 중요한 건 비프음으로 짧게.

→ 경고를 계층으로 나눠야 함

| 단계 | 기준 | 출력 |
|------|------|------|
| 즉각 위험 | 차량 8m 이내, 계단 1m, 칼·가위 3m | 음성 (즉시) |
| 일반 장애물 | 1m 이내 물체 | 비프음 |
| 정보성 안내 | 2m 이상 물체, 군중 | 음성 명령 시에만 |

---

### 2. 카메라로 거리 측정은 안 된다

단안 카메라 거리 추정은 오차가 크고 강사님도 그걸 아세요.
"1.2미터" 같은 수치를 말하는 게 과장처럼 보일 수 있어요.

→ 발표할 때는 솔직하게: "Depth V2로 상대 거리 추정, 정밀 측정은 한계가 있어요"
→ 음성 멘트는 수치 대신 상대 표현으로 바꾸기

| 기존 | 변경 |
|------|------|
| "약 1.2미터" | "가까이" |
| "약 2.5미터" | "조금 멀리" |
| "약 0.3미터" | "바로 앞" |

---

### 3. 타겟 유저를 명확히 해라

시각 장애 유형이 여러 가지인데 다 잡으려 하면 아무것도 못 해요.

시각 장애 유형:
- 시력 장애: 사물 식별력 저하
- 시야 장애: 터널 시야, 중심 시야 손실
- 색각 장애: 색맹·색약
- 광각 장애: 눈부심, 야맹증
- 굴절·조절 장애: 초점 문제

→ **전맹(완전 시각 장애)을 주 타겟으로** 설계
→ 다른 유형은 기능 끄기로 대응
→ 발표에서 "전맹 사용자를 주 타겟으로 설계했다"고 명확히 말하기

## 추가하고 싶은 기능

### 신호등 파란불 감지 ⭐

시각장애인분들이 신호등 앞에서 언제 건너야 할지 모르는 게 큰 두려움이에요.
현재 보조 기기는 신호등 앞에서 키를 누르면 불이 바뀔 때 진동으로 알려주는 방식인데,
저희는 카메라로 직접 감지할 수 있어요.

YOLO가 이미 신호등을 탐지하고 있어서,
탐지된 신호등 bbox 영역 안에서 상단(빨간불 위치)과 하단(초록불 위치)의 밝기를 비교하면
색상 판별이 가능해요. 빨간불 → 초록불로 바뀌는 순간을 잡아서
"신호등이 초록불로 바뀌었어요. 건너가세요." 안내.

구현 방향:
- YOLO `traffic light` 탐지 후 bbox crop
- HSV 색공간에서 초록 영역 비율 계산
- 이전 프레임 대비 초록 비율이 올라가면 → 전환 감지

바닥 전체 깊이 분산이 크면 "바닥이 고르지 않아요. 천천히 걸으세요."

### 낙하 물체·날아오는 물체 감지

프레임 간 물체 위치 변화 속도 추적.
빠르게 아래로 내려오거나 카메라 방향으로 접근하는 물체 → 즉시 경고.
떨어지는 물건, 날아오는 공 등.

### 음성 자동 시작

앱이 켜지면 먼저 "음성 안내를 시작할까요? 네 또는 아니오로 말씀해주세요." 출력.
버튼을 못 찾는 문제 해결.

| 사용자 조사 추가 | 시각장애인 실제 불편 사례 조사 | 🟢 발표 준비 |
