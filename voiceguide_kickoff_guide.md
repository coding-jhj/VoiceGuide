## 파트 1. 뭘 만드는지 (1분)

### 핵심 메시지
> "시각장애인이 말하면, 카메라가 보고, 음성으로 알려준다"

### 이것만 전달하면 됨
- 서비스 이름: **보이스가이드**
- 핵심 기능은 딱 3개
  - "앞에 뭐 있어?" → 장애물 안내
  - "컵 찾아줘" → 물건 찾기
  - "이거 뭐야?" → 물건 확인
- 완벽한 앱이 목표가 아니라, **이 3개가 작동하는 데모**가 목표

### 자연스럽게 말하는 흐름
1. 서비스 이름 말하기
2. 예시 3개를 입력 → 출력 형태로 보여주기
3. "이 3개만 제대로 만드는 게 목표"로 마무리

## 파트 3. 언제 어떻게 할지 (1분 30초)

### 핵심 메시지
> "7일차까지 데모 완성, 8일차부터 새 기능 추가 금지"

### 10일 흐름 (이것만 기억하면 됨)

| 구간 | 핵심 |
|---|---|
| 1~2일차 | 환경 세팅 + YOLO 테스트 + 기획 확정 |
| 3~4일차 | 방향/거리 판단 + TTS 연결 |
| 5~6일차 | STT 연결 + 시나리오 3개 통합 |
| **7일차** | **통합 데모 완성 ← 핵심 체크포인트** |
| 8일차 | 테스트 + 오류 수정 |
| 9~10일차 | PPT + 데모 영상 + 리허설 |

### 강조할 포인트 2개
1. **7일차 체크포인트**: 7일차까지 안 되면 범위 줄이기 (STT 빼거나, 물체 줄이거나)
2. **8일차 기능 추가 금지**: 욕심내다가 다 망하는 경우 방지

### 매일 회의 방식
- 15분만, 각자 3가지만 말하기
  1. 어제 한 일
  2. 오늘 할 일
  3. 막힌 점
- 회의 끝나면 바로 작업

## 파트 4. 기술 스택 (1분)

### 핵심 메시지
> "학습 없이 가져다 쓰는 것만, 욕심내면 망한다"

### 쓸 기술 (6개)

| 역할 | 기술 | 핵심 |
|---|---|---|
| 객체 탐지 | YOLO11n | 설치 한 줄, 학습 불필요, 80종 인식 |
| 방향 판단 | 직접 계산 | 코드 5줄, 라이브러리 없음 |
| 거리 판단 | 직접 계산 | 박스 크기로 가까움/멀리 판단 |
| TTS | gTTS | 한국어 자연스러운 음성 출력 |
| STT | SpeechRecognition | 어려우면 버튼으로 대체 가능 |
| UI | Gradio | 파이썬으로 웹 화면 자동 생성 |

### 절대 건드리면 안 되는 것 (이유 포함)
- Depth Anything → 거리 판단을 박스 크기로 대체 가능
- Grounding DINO → 키워드 매칭으로 대체 가능
- YOLO 파인튜닝 → 데이터 수집 + 학습만 5일
- Flutter/Android 앱 → 10일 안에 불가

> 발표에서는 "현재는 Python 데모, 실제 서비스는 Android 앱 전환 예정"이라고 하면 충분해요.

## 파트 5. 역할 분담 (1분)

### 핵심 메시지
> "6명이 3팀으로, 작업량이 비슷하게"

### 역할 구성

| 팀 | 인원 | 담당 |
|---|---|---|
| **구현 A** | 1명 | YOLO11n + 객체 탐지 + 방향/거리 판단 |
| **구현 B** | 1명 | TTS + STT + 의도 파악 (키워드 매칭) |
| **구현 C** | 1명 | 시나리오 3개 통합 + Gradio UI |
| **기획/자료** | 2명 | 자료조사 + PPT + 발표 대본 + 발표 |
| **테스트/데모** | 1명 | 테스트 설계 + 결과 기록 + 데모 영상 |

### 구현팀이 이렇게 나뉘는 이유
- A와 B가 각자 모듈을 독립적으로 만들고
- C가 그 두 모듈을 하나로 합치는 구조
- 서로 기다리지 않고 동시에 진행 가능

### 발표자
- 기획/자료팀 2명 중 1명
- 자료를 직접 만든 사람이 제일 잘 설명할 수 있기 때문

### 말하는 순서
1. 표 보여주면서 역할 설명
2. 구현팀 3개로 나눈 이유 짧게 설명
3. "하고 싶은 역할 있으면 말해주세요"로 넘기기
4. 침묵이면 먼저 지원자 물어보기

## 예상 질문 대비

**Q. 앱으로 만들면 안 되나요?**
Android 앱은 Kotlin을 새로 배우면서 10일 안에 완성하기 불가능해요. Python 데모로 만들고 "앱 전환 예정"이라고 하면 심사위원도 충분히 납득해요.

**Q. YOLO 파인튜닝 하면 더 정확하지 않나요?**
맞는데 데이터 수집부터 학습까지 최소 5일이에요. 기본 모델로도 컵, 의자, 가방은 잘 인식돼요. 데모에 필요한 물체만 골라 쓰면 충분해요.

**Q. STT가 잘 안 되면 어떻게 해요?**
버튼으로 대체해요. 발표에서 "STT 적용 시 이 버튼이 음성 명령으로 대체됩니다"라고 설명하면 돼요. 심사 점수에 큰 영향 없어요.

**Q. 공모전은 언제 준비해요?**
부트캠프 끝나고 바로 시작해요. 6월 1일까지니까 여유 있어요. 지금은 발표 1등에만 집중해요.
