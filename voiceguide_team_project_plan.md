# 보이스가이드 팀 프로젝트 기획서

## 1. 프로젝트 목표

시각장애인 또는 저시력 사용자가 스마트폰 카메라와 음성 안내를 통해 주변 상황을 더 쉽게 파악하도록 돕는 보조 앱을 만든다.

이 앱은 사용자가 화면을 직접 보거나 복잡하게 조작하지 않아도, 음성 명령과 카메라 인식을 통해 다음 문제를 해결하는 것을 목표로 한다.

- 주변 장애물을 탐지하고 방향과 거리를 음성 또는 진동으로 알려준다.
- 사용자가 찾고 싶은 물건을 말하면 카메라로 탐지해 방향을 알려준다.
- 사용자가 손에 든 물건이나 카메라 중앙의 물건을 음성으로 알려준다.
- 사용자의 상황에 따라 안내 길이와 말투를 조절한다.

핵심 방향은 단순히 객체 탐지 기능을 만드는 것이 아니라, **STT와 TTS를 활용해 시각 정보 접근이 어려운 사용자의 실제 불편을 해결하는 서비스**를 설계하는 것이다.

---

## 2. 추천 서비스명

### 보이스가이드

부제:

> 시각장애인을 위한 음성 기반 주변 인지 보조 서비스

이 이름을 쓰면 서비스 목적이 명확하게 드러난다.

- Voice: 사용자의 음성 명령
- Guide: 주변 상황을 안내하는 보조자
- 시각장애인/저시력 사용자에게 필요한 정보만 음성으로 전달

---

## 3. 주제 선정 이유

시각장애인과 저시력 사용자는 일상생활에서 다음과 같은 어려움을 겪을 수 있다.

- 낯선 실내 공간에서 앞에 무엇이 있는지 알기 어렵다.
- 의자, 책상, 문, 사람, 계단 같은 장애물을 빠르게 파악하기 어렵다.
- 원하는 물건이 어디 있는지 찾기 어렵다.
- 손에 든 물건이 맞는지 확인하기 어렵다.
- 스마트폰 화면 중심 서비스는 사용하기 불편할 수 있다.
- 주변 사람에게 계속 도움을 요청해야 하는 부담이 있다.

따라서 이 프로젝트는 시각 정보를 음성 정보로 바꾸어 사용자가 더 독립적으로 이동하고 물건을 찾을 수 있도록 돕는 것을 목표로 한다.

## 4. 사용자 분석

### 주요 사용자

시각장애인 또는 저시력 사용자

### 사용자 특성

- 시각 정보 확인이 어렵거나 제한적이다.
- 스마트폰 화면을 직접 보는 방식보다 음성 안내가 더 편리하다.
- 복잡한 버튼 조작보다 말로 명령하는 방식을 선호할 수 있다.
- 안내가 너무 길면 오히려 이동 중 불편할 수 있다.
- 위험 상황에서는 짧고 강한 안내가 필요하다.

### 사용 상황

- 교실, 복도, 집, 도서관, 카페 같은 실내 공간
- 책상 위나 주변에서 물건을 찾는 상황
- 낯선 공간에서 장애물을 피해야 하는 상황
- 손에 든 물건이 무엇인지 확인해야 하는 상황

### 불편함 또는 문제 상황

- 앞에 장애물이 있는지 알기 어렵다.
- 물건이 왼쪽에 있는지 오른쪽에 있는지 파악하기 어렵다.
- 화면을 보며 앱을 조작하기 어렵다.
- 기존 객체 인식 서비스는 시각 결과만 보여주는 경우가 많다.
- 안내가 너무 자세하거나 너무 늦으면 실제 상황에서 도움이 적다.

### 기존 방식의 한계

- 주변 사람에게 도움을 요청해야 한다.
- 지팡이는 낮은 장애물이나 일부 물체는 확인할 수 있지만, 물건의 종류나 위치를 구체적으로 알려주지는 못한다.
- 일반 카메라 앱은 화면 확인이 필요하다.
- 객체 탐지 앱은 탐지 결과를 보여주더라도 사용자가 화면을 보기 어려우면 활용성이 떨어진다.

## 5. 문제 정의

### 한 문장 문제 정의

> 시각장애인과 저시력 사용자는 기존의 화면 중심 정보 제공 방식으로 인해 주변 장애물, 물건 위치, 손에 든 물체를 즉시 파악하기 어려우므로, 이를 음성 명령과 음성 안내로 해결할 수 있는 주변 인지 보조 서비스가 필요하다.

### 문제 정의의 핵심 질문

- 누가 문제를 겪고 있는가?
  - 시각장애인 또는 저시력 사용자

- 무엇이 불편한가?
  - 주변 장애물, 물건 위치, 손에 든 물체를 시각적으로 확인하기 어렵다.

- 왜 이것이 문제인가?
  - 이동 중 안전 문제와 물건 탐색의 어려움이 생긴다.
  - 독립적인 생활과 이동에 제약이 생긴다.

- 현재 방식으로는 왜 해결이 어려운가?
  - 화면 중심 앱은 시각 정보 확인을 전제로 한다.
  - 주변 사람의 도움에 의존해야 할 수 있다.
  - 기존 보조 도구는 물체의 종류와 방향까지 자세히 알려주기 어렵다.

## 7. 솔루션 기획

### 서비스명

보이스가이드

### 서비스 목적

시각장애인 또는 저시력 사용자가 스마트폰 카메라와 음성 명령만으로 주변 장애물, 찾는 물건, 손에 든 물체를 파악하도록 돕는다.

### 핵심 기능

1. 음성 명령 입력
   - 사용자가 말로 앱을 조작한다.
   - 예시:
     - “앞에 뭐 있어?”
     - “컵 찾아줘”
     - “이거 뭐야?”
     - “짧게 말해줘”
     - “자세히 설명해줘”

2. 카메라 기반 객체 인식
   - 스마트폰 카메라로 주변 물체를 인식한다.
   - 예시 객체:
     - 의자
     - 책상
     - 컵
     - 가방
     - 문
     - 사람
     - 계단

3. 방향 및 거리 안내
   - 탐지된 물체의 위치를 사용자에게 이해하기 쉬운 말로 전달한다.
   - 방향:
     - 왼쪽
     - 오른쪽
     - 정면
   - 거리:
     - 가까움
     - 중간
     - 멀리
   - 예시:
     - “왼쪽 앞에 컵이 있습니다.”
     - “정면 가까이에 의자가 있습니다.”

4. TTS 음성 안내
   - 탐지 결과를 음성으로 출력한다.
   - 사용자 상황에 따라 안내 길이와 말투를 조절한다.

5. STT 기반 모드 전환
   - 사용자의 말을 분석해 어떤 기능이 필요한지 자동으로 판단한다.
   - 예시:
     - “찾아줘” → 물건 찾기 모드
     - “이거 뭐야?” → 물체 설명 모드
     - “앞에 뭐 있어?” → 장애물 확인 모드
     - “짧게 말해줘” → 짧은 안내 모드

6. 진동 피드백
   - 장애물이 가까울수록 진동을 강하게 하거나 빠르게 한다.
   - 청각 안내가 어려운 상황에서도 위험을 전달할 수 있다.

## 8. 보완 및 추가 기능

### 8-1. 위험도 우선 안내

단순히 물체를 나열하지 않고, 사용자의 안전에 중요한 정보를 먼저 알려준다.

예시:

- 일반 안내:
  - “앞에 의자와 책상이 있습니다.”

- 위험도 우선 안내:
  - “정면 1m 앞에 의자가 있습니다. 오른쪽으로 피하세요.”

위험도 기준:

- 사용자와 가까운 물체
- 이동 경로를 막는 물체
- 충돌 가능성이 높은 물체
- 사람, 계단, 문턱 등 안전과 관련 있는 요소

### 8-2. 상황별 음성 모드

사용 상황에 따라 모드를 나누면 서비스가 더 구체적으로 보인다.

1. 실내 이동 모드
   - 장애물, 문, 책상, 의자 등을 안내한다.

2. 물건 찾기 모드
   - 사용자가 말한 물건을 찾아 방향을 알려준다.
   - 예: “컵 찾아줘” → “컵은 왼쪽 앞에 있습니다.”

3. 물건 확인 모드
   - 손에 든 물건이나 카메라 중앙의 물건을 설명한다.
   - 예: “카메라 중앙의 물건은 텀블러입니다.”

4. 긴급 모드
   - “도와줘”라고 말하면 보호자 연락, 현재 위치 안내, 도움 요청 메시지 같은 시나리오를 제안할 수 있다.
   - 실제 구현이 어렵다면 발표에서는 확장 기능으로 제시한다.

### 8-3. 개인화 TTS

사용자는 상황에 따라 다른 안내 방식을 원할 수 있다.

안내 방식 예시:

- 짧은 안내:
  - “정면 장애물”

- 자세한 안내:
  - “정면 가까운 곳에 의자가 있습니다. 오른쪽으로 이동하세요.”

- 긴급 안내:
  - “멈추세요. 바로 앞에 장애물이 있습니다.”

이 기능은 사진 속 요구사항의 **개인화 TTS-STT 모델링 및 성능 개선하기**, **입력값에 따라 음성이 변화되는 나만의 TTS-STT 모델 구현하기**와 잘 연결된다.

## 9. TTS-STT 활용 방식

이 프로젝트에서 STT와 TTS는 단순히 붙이는 기술이 아니라, 서비스 문제 해결의 중심 역할을 한다.

### STT 활용

사용자의 음성을 텍스트로 변환한다.

예시:

- 사용자 음성: “앞에 뭐 있어?”
- STT 결과: 앞에 뭐 있어?
- 의도 분석: 주변 장애물 확인

### TTS 활용

앱의 분석 결과를 음성으로 출력한다.

예시:

- 분석 결과:
  - 객체: 의자
  - 방향: 정면
  - 거리: 가까움
- TTS 출력 문장:
  - “정면 가까이에 의자가 있습니다. 오른쪽으로 이동하세요.”

### 입력값에 따른 음성 변화

사용자의 요청 또는 상황에 따라 음성 출력이 달라진다.

| 입력 조건 | 출력 방식 |
|---|---|
| “짧게 말해줘” | 짧고 빠른 안내 |
| “자세히 설명해줘” | 자세한 설명형 안내 |
| 위험 물체가 가까움 | 강하고 짧은 긴급 안내 |
| 물건 찾기 요청 | 방향 중심 안내 |
| 물건 확인 요청 | 물체 이름 중심 안내 |

## 10. 전체 파이프라인 구조도

```text
사용자 음성 입력
        ↓
STT 변환
        ↓
명령 의도 분석
        ↓
모드 선택
장애물 탐지 / 물건 찾기 / 물체 설명
        ↓
카메라 이미지 분석
        ↓
객체명 + 방향 + 거리 추정
        ↓
안내 문장 생성
        ↓
개인화 TTS 출력
        ↓
음성 또는 진동 피드백
```

### 단계별 설명

1. 사용자 음성 입력
   - 사용자가 말로 명령한다.

2. STT 변환
   - 음성을 텍스트로 바꾼다.

3. 명령 의도 분석
   - 사용자가 원하는 기능을 판단한다.

4. 모드 선택
   - 장애물 탐지, 물건 찾기, 물체 설명 중 하나를 선택한다.

5. 카메라 이미지 분석
   - 카메라 화면 속 객체를 탐지한다.

6. 객체명, 방향, 거리 추정
   - 탐지 결과를 사용자에게 필요한 정보로 정리한다.

7. 안내 문장 생성
   - 사용자의 모드에 맞게 문장을 만든다.

8. 개인화 TTS 출력
   - 짧게, 자세히, 긴급하게 등 상황별 음성 안내를 제공한다.

9. 음성 또는 진동 피드백
   - 음성 안내와 함께 위험 상황에서는 진동도 제공한다.

## 11. 대표 데모 시나리오

### 시나리오 1. 장애물 회피

사용자 상황:

- 사용자가 교실 안을 걷고 있다.
- 정면에 의자가 있다.

음성 입력:

> 앞에 뭐 있어?

앱 처리:

- STT로 음성 인식
- 카메라로 정면 물체 탐지
- 의자가 가까운 위치에 있음을 판단

출력:

> 정면 가까이에 의자가 있습니다. 오른쪽으로 이동하세요.

### 시나리오 2. 물건 찾기

사용자 상황:

- 사용자가 책상 위의 컵을 찾고 있다.

음성 입력:

> 컵 찾아줘

앱 처리:

- STT로 “컵”과 “찾아줘” 인식
- 물건 찾기 모드 실행
- 카메라 화면에서 컵 탐지
- 컵의 방향 판단

출력:

> 컵은 왼쪽 앞에 있습니다.

### 시나리오 3. 물건 확인

사용자 상황:

- 사용자가 손에 든 물건이 무엇인지 알고 싶다.

- 방향은 화면을 3등분해서 왼쪽/정면/오른쪽으로 판단한다.

```text

오른쪽 영역 → “오른쪽에 있습니다”
```

거리 판단 예시:

```text
객체 박스 크기가 큼   → 가까움
객체 박스 크기가 중간 → 중간 거리
객체 박스 크기가 작음 → 멀리 있음
```

## 18. 1등을 위한 차별화 포인트

이 프로젝트는 단순히 “카메라로 물체를 인식하는 앱”이 아니다.

핵심 차별점은 다음과 같다.

1. 사용자 문제에서 출발한다.
   - 기술을 먼저 정한 것이 아니라, 시각 정보 접근이 어려운 사용자의 pain point에서 기능을 도출했다.

2. STT와 TTS가 서비스 흐름의 중심이다.
   - STT는 명령 입력을 담당한다.
   - TTS는 상황 안내를 담당한다.
   - 단순 기술 시연이 아니라 서비스 맥락 안에서 작동한다.

3. 안내가 상황에 따라 달라진다.
   - 위험 상황에서는 짧고 강하게 안내한다.
   - 물건 찾기에서는 방향 중심으로 안내한다.
   - 물건 확인에서는 물체 이름 중심으로 안내한다.

4. 접근성 관점이 명확하다.
   - 화면 조작을 줄인다.
   - 음성 중심 인터페이스를 사용한다.
   - 진동 피드백까지 고려한다.

5. 데모가 명확하다.
   - 장애물 회피
   - 물건 찾기
   - 물건 확인

발표에서 강조할 문장:

> 저희 서비스는 단순 객체 탐지가 아니라, 사용자의 음성 의도와 주변 상황을 결합해 필요한 정보만 음성으로 안내하는 TTS-STT 기반 접근성 서비스입니다.

## 19. 발표 시 주의할 점

사진 속 유의사항에 맞춰 다음을 반드시 지킨다.

### 피해야 할 방식

- 기능만 나열하는 발표
- 사용자 없이 기술만 강조하는 구성
- 문제 정의 없이 바로 구현 설명으로 들어가는 방식
- 결과 비교나 테스트 없이 “작동한다” 수준으로 마무리
- STT/TTS를 단순히 붙였다고만 설명하는 방식

### 지향해야 할 방식

- 사용자가 누구인지 먼저 말한다.
- 어떤 상황에서 불편한지 설명한다.
- 왜 이 서비스가 필요한지 제시한다.
- 고객 여정 지도와 pain point를 보여준다.
- 그 pain point를 해결하기 위해 STT/TTS가 왜 적합한지 설명한다.
- 파이프라인과 데모 결과를 함께 보여준다.
- 테스트 결과를 표로 정리한다.
